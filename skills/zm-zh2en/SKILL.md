# zm-zh2en

## 核心合同

- **输入**：用户当前会话中直接粘贴的中文文本；不接受本地文件路径、URL 或网页抓取。
- **方向**：固定中译英（zh-CN → en），不支持其他语言对。
- **输出**：翻译结果直接输出到当前会话窗口，不保存到本地文件。
- **流程**：完整四轮模式下必须经历分析、草稿、审校、定稿全部步骤；默认快速三轮将审校简化为草稿后的快速自查。
- **范围**：支持短段落、长文章、技术文档、论文摘要等多种长度。建议单次输入不超过 8000 中文字符，超长文本可能分批处理或提示拆分。

## 安全与隐私

- 用户输入的文本仅用于当前会话的翻译处理，不保存在本地文件，也不额外上传至除底层 AI 模型服务外的第三方。
- 翻译过程需经过底层模型服务处理，建议避免在翻译模式中输入密码、密钥、身份证号等高度敏感信息。

## 会话模式

本 skill 被调用后，当前会话进入中译英模式。进入时应输出简短状态提示（如已进入中译英模式，粘贴中文文本即可自动翻译）。退出时也应确认状态。在此模式下：

- **自动触发**：当用户粘贴或输入的文本满足以下条件之一时，视为中文输入并自动执行翻译流程：
  - CJK 统一表意文字与全角标点合计超过 20 个；
  - 包含至少一个以中文标点（。！？）结尾的完整语句；
  - 连续 CJK 统一表意文字超过 15 个（兜底无结尾标点的段落）。
  少于 20 个字符的简短英文指令，或少于 10 个字符的中文内容，按正常对话处理，不触发翻译。
- **对话识别**：简短英文指令（如 "exit"、"revise the third paragraph"）不触发翻译，按正常对话处理。
- **退出模式**：用户明确说"退出翻译""停止""结束"或类似表达，或输入英文"exit"、"stop"、"end"、"quit"、"done"时，退出中译英模式，恢复正常对话。
- **模式生命周期**（受平台限制）：翻译模式持续至用户显式退出或当前会话结束。会话中断、新线程或长时间不交互不会自动保留模式状态。
- **模式冲突**（受平台限制）：若用户在翻译模式中调用其他 skill，后调用的 skill 覆盖当前模式。
- **模式内调整**：用户在翻译输出后提出的修改意见（如"术语改用 XX""第三段再润色"），视为对最近一次译文的反馈，直接重新处理。

## 翻译流程（内部多轮）

各轮为内部思考过程，不中断用户、不要求逐轮确认，最终直接输出定稿译文。

### 第一轮：分析

- 识别文本类型，提取关键术语，预判翻译难点。
- 确定语气和风格，形成翻译策略。

### 第二轮：草稿

- 基于分析结论产出完整英文草稿。
- 优先写出英文里自然成立的句子，避免逐句硬译。
- 保持原文结构（段落、列表、代码块等），术语前后一致。

### 第三轮：审校

- 逐段检查译文的准确性、完整性和一致性。
- 排查术语误译、长句拗口、语气失当等问题。
- 记录修改要点，供定稿轮修正。

### 第四轮：定稿

- 整合审校修正，产出最终译文。
- 直接向用户输出译文，可附一句极简短说明，不展开各轮细节。

## 交互约定

- 用户可在输出后提出修改意见，skill 即时响应并重新处理。默认重新走完整分析→草稿→定稿流程；若用户明确要求只修改局部（如"只改第三段"），则仅针对指定部分重新处理。
- 默认执行快速三轮（分析 → 草稿 → 定稿，审校简化为快速自查）。
- 用户明确要求"完整翻译""带审校"或"完整四轮"时，执行完整四轮。
- 用户要求跳过某轮时，尊重用户决策。分析轮和审校轮可跳过，草稿轮和定稿轮必须保留，以确保输出质量。注意：跳过分析轮可能导致术语识别不完整；跳过审校轮可能遗漏一致性错误和术语误译。

## 质量要求

- **自然优先**：译文优先写出英文里自然成立的句子。
- **结构对应**：段落、列表、代码块等结构尽量对应原文。
- **术语一致**：专业术语前后统一，必要时提供译名对照。
- **事实准确**：事实、数据、日期、专有名词与原文一致。
- **语气匹配**：译文语气与原文一致。

## 失败口径

| 状态码 | 含义 |
|---|---|
| `empty_input` | 用户未提供待翻译文本 |
| `unsupported_language` | 输入文本明显不是中文 |
| `user_abort` | 用户主动中止 |
| `mixed_language` | 输入为混合中英文，按主语言判断处理：中文字符占比 ≥ 50% 时按中文处理并翻译；否则视为非中文输入，返回 `unsupported_language` |
| `too_short` | 输入过短，未达到自动翻译触发阈值 |
| `mode_already_active` | 用户已在翻译模式中再次调用本 skill |
| `mode_not_active` | 用户未调用 skill 即要求翻译 |
| `provider_error` | 底层模型服务异常 |

正常完成时返回 `success`（翻译完成）。
