# zm-en2zh

## 核心合同

- **输入**：用户当前会话中直接粘贴的英文文本；不接受本地文件路径、URL 或网页抓取。
- **方向**：固定英译中（en → zh-CN），不支持其他语言对。
- **输出**：翻译结果直接输出到当前会话窗口，不保存到本地文件。
- **流程**：默认执行分析 → 草稿 → 审校 → 定稿四轮。用户明确要求快速/跳过某轮时，按用户指令降级执行，并在回复开头简要说明实际执行轮次。
- **范围**：支持短段落、长文章、技术文档、论文摘要等多种长度。建议单次输入不超过约 3000 个英文单词或 4000 个模型 token，超长文本建议分段处理。

## 调用方式

- **Codex CLI**：输入 `/zm-en2zh` 触发；入口配置见 `agents/openai.yaml`。
- **Claude Code**：通过 Skill 工具调用 `zm-en2zh`。
- **其他生态**：在支持 Skill 工具调用的生态中，直接调用 `zm-en2zh`。
- 本 skill 按当前消息独立处理，不依赖跨轮持久化状态。如需继续翻译，请重新调用并粘贴新文本。

## 翻译流程

翻译在内部按分析 → 草稿 → 审校 → 定稿推进，不中断用户、不要求逐轮确认，最终直接输出定稿译文。

- **分析**：识别文本类型、语气与关键术语，预判翻译难点。
- **草稿**：产出完整中文译文，优先中文自然表达，保持原文结构对应。
- **审校**：对照原文检查准确性、术语一致性与语气匹配；完整审校形成修正清单，快速模式下至少检查术语一致性、数字/专有名词、明显语病。
- **定稿**：整合修正，输出最终译文；可在译文前附一句极简说明（如"已审校修正术语与长句"），不展开各轮细节。

## 交互约定

- 默认执行完整四轮（分析 → 草稿 → 审校 → 定稿）。
- 用户明确要求"快速翻译""跳过审校"或"跳过某轮"时，按用户指令降级执行，并在回复开头简要说明实际执行轮次。
- 用户如需修改译文，请在当前消息中直接附上需要调整的译文片段和修改要求，skill 将基于当前上下文即时响应并重新处理。
- 用户可随时通过发送"退出""停止""结束"或"exit""stop""end"终止当前翻译请求。

## 质量要求

- **自然优先**：译文优先写出中文里自然成立的句子。
- **结构对应**：段落、列表、代码块、引用等结构尽量对应原文。
- **术语一致**：专业术语前后统一，必要时提供译名对照；长文本翻译可在译文末尾附「术语译名对照」小节。
- **事实准确**：事实、数据、日期、专有名词与原文一致。
- **语气匹配**：译文语气与原文保持一致。

## 边界口径

当输入不满足翻译条件时，按以下方式回复：

- **空输入**：「请粘贴待翻译的英文文本。」
- **明显非英文**：「当前 skill 仅支持英译中（en → zh-CN），请粘贴英文文本。如需翻译本地文件，请使用 `zm-translate-from-files`。」
